# Lympha Evidence Engine — бэкенд

Планировщик + API, который наполняет ленту, дайджест и темы приложения Лимфа
готовыми подборками статей из **Europe PMC**. Реализует ТЗ из технического
описания (v1.0, июнь 2026).

## Что делает

1. **Сбор** (раз в сутки, 04:00) — опрашивает Europe PMC по запросу для каждой
   из 7 специальностей за окно последних 30 дней.
2. **Отбор** — нормализует, отсекает ранее показанные статьи (антидубли),
   ранжирует по уровню доказательности → цитируемости → свежести.
3. **Выдача** — сохраняет готовый снимок и отдаёт клиенту через свой API
   в формате из раздела 6 ТЗ.

## Стек

- **Node.js** (≥18) + **Express** — API
- **node-cron** — планировщик
- **JSON-на-диске** — хранилище снимков (`data/`). Ноль нативных зависимостей.
  Когда понадобится персонализация по пользователям — меняется только
  `src/lib/storage.js`, интерфейс остаётся прежним.

## Установка и запуск

```bash
npm install
npm run collect      # разовый сбор вручную (нужен доступ в интернет к ebi.ac.uk)
npm start            # поднять API + планировщик на порту 3000
```

### Переменные окружения

| Переменная | Назначение | Обязательна |
|---|---|---|
| `PORT` | порт сервера (по умолч. 3000) | нет |
| `ANTHROPIC_API_KEY` | перевод заголовков на русский | нет (без неё отдаются оригиналы) |
| `ADMIN_TOKEN` | защита ручного триггера `/api/admin/collect` | нет |

## Эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/feed` | лента; фильтр `?specialty=implant` |
| GET | `/api/digest` | дайджест топ-3; фильтр `?specialty=` |
| GET | `/api/article/:pmid` | содержимое статьи (прокси к Europe PMC) |
| GET | `/api/article/:pmid/fulltext` | полный текст OA-статьи (секции) |
| GET | `/api/guidelines` | клинические руководства; фильтр `?specialty=` |
| GET | `/api/health` | статус + время последней сборки |
| GET | `/api/admin/collect?token=…` | ручной запуск сбора |

## Полный текст (Open Access)

Статья из ответа `/api/article/:pmid` теперь содержит:
- `pmcid` — идентификатор для полного текста
- `fullTextAvailable` — `true`, если статья OA и текст доступен
- `isOpenAccess` — флаг открытого доступа

Эндпоинт `/api/article/:pmid/fulltext` возвращает массив секций:
```json
{ "pmid": "...", "pmcid": "PMC...", "fullText": [ { "heading": "Methods", "text": "..." } ] }
```
Если статья не OA — `fullText: null`, `reason: "not_open_access"`. Клиент в этом
случае показывает аннотацию. **Граница легальности:** полный текст отдаётся
только для статей открытого доступа.

## Клинические руководства

Курируемый реестр официальных руководств (`src/data/guidelines.js`): ADA, AAP,
AAE, EFP, ITI, ACP, AAO, AAOMS, AAPD, Минздрав РФ, СтАР. Два типа записей:
- `type: 'journal'` — есть в PubMed (поле `pmid`), открывается как обычная статья
- `type: 'web'` — на сайте организации (поле `url`)

`/api/guidelines?specialty=perio` возвращает профильные руководства + общие
(ADA, Минздрав, СтАР показываются для всех специальностей).

**Важно:** реестр курируется вручную. Перед продакшеном команда проверяет
актуальность ссылок, годов и PMID — организации обновляют документы.

## Специальности

`implant`, `endo`, `perio`, `prosth`, `ortho`, `surgery`, `child`
— термины и запросы заданы в `src/lib/specialties.js`.

## Тесты

```bash
node test/pipeline.test.js   # 14 проверок: ранжирование, антидубли, дайджест, формат
```

## Деплой

Подойдёт любой Node-хостинг (Railway, Render, Fly.io, VPS). Для постоянной
работы планировщика нужен «живой» процесс (не serverless). На Railway/Render
достаточно `npm start` как start-команды и persistent volume для папки `data/`.

## Замечание про песочницу разработки

Код протестирован на мок-данных Europe PMC (формат ответа `resultType=core`).
Живые сетевые вызовы к `ebi.ac.uk` и `api.anthropic.com` проходят с обычного
сервера без изменений в коде — в среде, где собирался прототип, эти домены
были недоступны, поэтому проверялась логика, а не сетевой слой.

## Структура

```
src/
  server.js              API + cron
  jobs/collect.js        конвейер сбор→отбор→выдача
  lib/
    specialties.js       маппинг специальностей, типы доказательности
    europepmc.js         клиент Europe PMC (поиск, статья, полный текст)
    fulltext.js          парсер JATS XML → читаемые секции
    ranking.js           скоринг, дайджест, "почему важно"
    translate.js         перевод заголовков через Claude API
    storage.js           снимки + антидубли (JSON-на-диске)
  data/
    guidelines.js        курируемый реестр клинических руководств
test/
  pipeline.test.js       проверка конвейера на мок-данных
  fulltext.test.js       проверка парсера полного текста
data/                    снимки (создаётся при первом сборе)
```
